# プロジェクト記憶更新スキル (Project Memory Skill)

## 目的
複数回のチャットやセッションを跨いでプロジェクトの前提条件、最新のアーキテクチャ上の決定事項、直近の作業内容を引き継ぐための「長期記憶」を形成する。

## 実行フロー
このスキルを実行する際は、以下のステップに沿って `.agents/PROJECT_MEMORY.md`（または指定の記憶ファイル）を更新してください。

### 1. 最新の記憶の確認
`view_file` ツール等のファイル読み込みツールを使用して、現在の `.agents/PROJECT_MEMORY.md` の内容を確認してください。

### 2. 今回の作業の要約
今回のやり取り・タスクによってどのような変更・議論が行われたかを日本語で要約してください。以下の情報を含めること：
- **対応日時/タイムスタンプ** (日付と大まかな時間)
- **達成したタスクや追加した機能**
- **対応したバグやその根本原因の記録**
- **新たに追加・変更された主要なファイルや依存関係**
- **次回への申し送り事項・未解決のエッジケース（存在すれば）**

### 3. ファイルへの追記
`replace_file_content` ツール等を使用して、`.agents/PROJECT_MEMORY.md` の **『最近の作業ログ (Recent Work Logs)』** セクションの一番上に最新の要約を追記してください。
過去のログが多すぎる場合（例えば20件以上）は、古いものを圧縮・削除して整理しても構いません。

### 4. 実行後の報告
記憶の書き込みが完了したら、ユーザーに対し「プロジェクトの記憶を更新しました」という旨と、記録した内容の簡単なサマリーを報告してタスクを終了してください。
