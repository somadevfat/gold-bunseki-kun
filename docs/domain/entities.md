# エンティティ定義 (Entities)

クリーンアーキテクチャの中心となる、技術（DBやAPI）に依存しない「核となるデータ構造」です。
※GoやPythonのクラス/構造体として実装されるイメージです。

## 1. PriceRecord (価格データ)

1分足のXAUUSDデータを表す構造体。

- `Timestamp` (日時): 記録された日時（UTC）
- `Open` (始値): その1分の開始価格
- `High` (高値): その1分の最高価格
- `Low` (安値): その1分の最安価格
- `Close` (終値): その1分の終了価格

## 2. Volatility (ボラティリティ)

特定の時間枠内で発生した「値幅」を表す構造体。

- `TimeWindow` (時間枠): 例「09:00 - 12:00（東京時間）」など
- `TargetDate` (対象日): 算出対象の日付
- `HighPrice` (期間最高値): 時間枠内で算出された「ZigZagの最高点（スイングハイ）」の価格
- `LowPrice` (期間最安値): 時間枠内で算出された「ZigZagの最下点（スイングロー）」の価格
- `Value` (ボラティリティ値): 上記 `HighPrice - LowPrice` の結果（ドル幅。これが真のボラティリティとなる）
- `Deviation` (逸脱度): 基準値に対してどれくらい乖離しているか（大・中・小など）
- `RelatedIndicatorID` (関連指標ID): この時間枠で発生した経済指標のID（※ここで経済指標Entityと紐付けます。指標がない時間帯もあるため空を許容）

## 3. EconomicIndicator (経済指標)

相場に影響を与えるイベントを表す構造体。

- `ID` (識別子): 一意のID
- `Name` (指標名): 例「米国 非農業部門雇用者数（NFP）」
- `ReleaseTime` (発表日時): イベントが発生する日時（UTC）
- `Importance` (重要度): 高・中・低など
- `Forecast` (独自の予想値): 外部APIには依存せず、システム内部（過去の経済指標データ等の統計）から独自に算出した予測値
- `Actual` (結果値): APIから取得した発表済みの実際の結果
- `CausedVolatilityID` (発生波のID): この指標発表によって発生した「特定の波（ボラティリティのID・またはZigZagPointのID）」

## 4. ZigZagPoint (転換点・スイング)

価格のノイズを排除し、相場の意味のある「高値」と「安値」を結んだ波の頂点と底を表す構造体（またはその配列）。

- `Timestamp` (日時): 転換点となった日時
- `Price` (価格): その時の価格
- `Type` (種類): `High` (スイングハイ) か `Low` (スイングロー) か

## 5. User (プレミアムユーザー)

AI予測の閲覧など、有料サブスクリプションに加入したユーザーだけを管理する構造体。（※基本機能はログイン不要・アカウント登録不要で全開放するため、システム上にユーザーとして存在＝有料会員となる）

- `ID` (識別子): 一意のID（Stripe等の決済IDと紐づく）
- `Name` (名前): お客様名
- `Email` (メールアドレス): ログインID、および重要なお知らせ送付用
- `Password` (パスワード): ハッシュ化されたパスワード
- `Role` (役割): `Admin`（管理者） または `PremiumUser`（有料会員）
- `SubscriptionValidUntil` (サブスク有効期限): 有料プランがいつまで有効かの日時（失効した場合は予測表示不可にダウンタイムさせる判定に使う）
